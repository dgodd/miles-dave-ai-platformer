# Platformer

A 2D side-scrolling platformer built entirely with code written by an AI language model (Claude).

**This is an experiment.** My son and I wanted to see how far we could get an LLM to build a complete, playable game just by describing what we wanted in plain English. Every line of code in this repository was generated by an AI — we just played the game, pointed out what was wrong, and asked for changes.

## How it works

Jump and run across platforms, avoid spike pits, and dodge crawling babies. All sprites are drawn with code — rectangles, circles, and triangles — so there are no image assets anywhere.

## Controls

| Key | Action |
|---|---|
| A / Left | Move left |
| D / Right | Move right |
| Space / W / Up | Jump |
| R | Reset |

## Running it

```sh
cargo run
```

Requires Rust. Built with `macroquad`.

## Release builds

After every commit a `post-commit` hook automatically compiles an
optimised binary and drops it into `release/`. To enable it:

```sh
git config core.hooksPath hooks
```

On macOS the native binary is produced automatically. Windows
cross-compilation requires the mingw target:

```sh
rustup target add x86_64-pc-windows-gnu
# macOS users also need:
# brew install mingw-w64
```

## The experiment

We started with a blank directory and one instruction: *"create the start of a platformer game using rust."* From there, everything — the physics, the level design, the dog character, the spike pits, the baby enemies, the death messages — came from telling the AI what we wanted next. We never wrote a single line of code ourselves.

Some highlights from the conversation:

- *"when you fall off you fall forever. please add spikes so you die to them instead"*
- *"using Dog.jpeg turn this image into a walking, jumping sprite for the character"* (the AI couldn't see the photo, so it invented an original dog character inspired by it)
- *"i changed my mind. use Dog.jpeg as inspiration for all the character sprites"*
- *"the tail and head are not attached to the body"* (and again, and again until it looked right)
- *"make some enemies that look like babies crawling around the map. if they catch you the death screen says 'The baby pulled your tail'"*

## What the AI wrote

~700 lines of Rust across a single file (`src/main.rs`), covering:

- Player physics (gravity, jumping, coyote time, variable jump height)
- Platform collision (axis-separated AABB)
- Camera system that follows the player
- Animated dog sprite with idle/walk/jump states
- Crawling baby enemies with patrol AI
- Spike pits with triangular teeth
- Death screen with custom messages
- All sprite art drawn procedurally with shapes

## Lessons

The AI is surprisingly good at game development when you treat it like a very eager, occasionally confused collaborator. It makes mistakes — sometimes the same one repeatedly — but it never gets tired of fixing them.

— Dave and Miles
